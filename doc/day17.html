Chapter 1. Day 17

Pushing Users to Heroku & Migration

Procedure 1.1.  Production level DB Migration

  1. Make sure all changes have been pushed to production (Heroku) $ git push heroku master

  2. Go to the app where it lives online at http://omr-kencortez.herokuapp.com/

    Tap the "Sign in" button and observe an error

  3. Open the console and run $ heroku logs --tail

    Search for the error in the stack trace

    If the error is not immediately apparent, just reproduce the error and watch the stack trace as it spews

  4. Heroku uses "pg" Postgress for its Database

    Need to run $ heroku rake db:migrate

    Migrations that we ran in development were not part of production

    Next time run $ heroku run rake db:migrate

    This command is harmless to run multiple times obviously

  5. Actually sign up for the application live and observe that it now works after running the migration in production
