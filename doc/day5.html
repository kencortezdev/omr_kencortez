Chapter 1. Day 5

Getting GitHub Set Up

Procedure 1.1.  Starting GitHub

  1. Create a GitHub

  2. We will need to SSH into our GitHub

    Create a key on our computer and give that to the GitHub preferences for secure access

    When we ran Rails Installer, the SSH key should have already been created for us

  3. See the next procedure for how to generate the SSH key and use with GitHub

Procedure 1.2.  Generating an SSH key on Windows

Note: Windows should have created a new SSH key when I ran Rails Installer. However, after a brief search, I couldn't find it so I am creating a new one to use with GitHub

  1. Open terminal and change to the root of the project folder

  2. Enter the command: $ ssh-keygen -t rsa -b 4096 -C "kenmcortez@gmail.com

  3. The prompt will ask for a file to save, but just hit ENTER and use the default

  4. Enter the passphrase (and remember it because we'll need it again shortly)

Procedure 1.3.  Adding the SSH key to the ssh-agent

  1. In the project folder, enter in the command: $ eval "$(ssh-agent -s)"

  2. Now add the new SSH key to the ssh-agent by entering: $ ssh-add ~/.ssh/id_rsa

Procedure 1.4.  Adding the SSH key to the GitHub account

  1. Copy the SSH key to the clipboard by entering: $ clip < ~/.ssh/id_rsa.pub

  2. Got to your GitHub account > "Settings" > "SSH and GPG keys"

  3. Add a meaningful title to what the key is for

  4. Enter the key and click Add

  5. Enter that password one more time that you should have remembered

Procedure 1.5.  Create a New GitHub project

  1. Project name is the same as the name of the root folder: omr_kencortez

    Do not initialize with a README

  2. Quick setup and choose the option for an existing repository

    Make sure that the SSH option is selected

  3. Enter the command: $ git init in the root of the project directory

  4. Type in and enter: $ git remote add origin git@github.com:kencortezdev/omr_kencortez.git

  5. And finally enter: $ git push -u origin master

    After this push, you should see the project up on GitHub

Procedure 1.6.  The GitHub Push Process

  1. Change the README in the root project folder to a markdown *.md format

  2. Delete the body and add some Markdown notes

  3. Run $ git status

    It will look like you've deleted a file and added another

  4. Run $ git add .

    Then run $ git status again

    Now files are staged to be added. One should show as "new file" and one should show as "deleted"

  5. Commit with git commit -am and specify that you changed the README in the notes

  6. This is still not up on GitHub; however, git hat the command to push up to GitHub

    The final command you must enter is $ git push

  7. The above section describes the three basic commands we will need

    $ git add .

    $ git commit -am "commit_notes"

    $ git push
