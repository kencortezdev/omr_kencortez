Chapter 1. Day 7

Rails uses an Model View Architecture (MVC). The model is out of the scope as of the moment. The View and the Controller are how users interact on the front end.

Procedure 1.1.  The Three Step Process for Creating Pages

In this section we will create an "about" page, but the same three step process can be applied generally to page creation

  1. Configuring the Controller

    Go to ./app/pages/controllers/pages_controller.rb

    Add the following to the class PagesController

    def about

    end

  2. Creating the Page File

    Next go to ./app/views/pages/

    Create a file in this directory called "about.html.erb"

    Add some well formatted HTML content to that page

    If you now try to navigate to localhost:3000/about you will get a routing error (this should be a hint as to what the final step is).

  3. Setting up the Routes

    Navigate to and open ./config/routes.rb

    Under Rails.application.routes.draw do, add this line of code to the ruby file:

    get "about" => "pages#about"

Procedure 1.2.  First Introduction to HTML and Embedded Ruby

  1. Modify the home file so that it contains the following under the header:

    <p>Sign up <%= link_to "here", "#" %>.</p>

    The line containing <%= link_to "here", "#" %>.</p> is embedded Ruby

    Clearly it is a Ruby Rails function "link_to" that takes two arguments: text of the link and the path

  2. Navigate to the "about" page and see that the HTML link equivalent has been generated using embedded Ruby.
