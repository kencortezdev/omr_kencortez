Chapter 1. Day 13

Pushing the App up to Heroku

Procedure 1.1.  Setting Up Heroku

  1. Create a Heroku account and login

    Keep this handy

  2. Download the "Heroku Toolbelt" at toolbelt.heroku.com

  3. Once Heroku is set up, run the command $ heroku help to make sure things are working

    Had some trouble with this is Cygwin, but it'll work in GitBash or PowerShell

  4. Check for SSH keys by typing $ heroku keys

    Add keys with $ heroku keys:add

  5. Heroku uses different database so the Gemfile will need to be modified

    Added groups: development and test and moved sqlite3 there

    This means we will be using the sqlite3 gem on our computer only

  6. Added another group to the gemfile for production

    Added the pg and 12factor gems for production (Heroku)

  7. Ran bundle install because we modified the gemfile

    If errors are encountered, you may have to run the command with an argument $ bundle install --without production

  8. Added the ruby version to the gemfile because of the bundle install warning

  9. Commit the changes

  10. Finally push the changes to Heroku with $ git push heroku master

    This means we want to push to Heroku and that we want to push the "Master" branch

  11. Renamed the app with $ heroku rename "omr-kencortez"

    Now the app is on the web at https://omr-kencortez.herokuapp.com
