Procedure 1.1.  Bootstrapping the Project

Again, this is in now way meant to be comprehensive. The doc is now basically a bullet point log of what was accomplished on a given day to serve as a reference.

  1. Added a "Container" to the app

    This was done in ./views/layouts/application.html.erb

  2. Create a header at the "layouts level" by creating the file _header.html.erb

    The underscore is quite important and basically means that this is file is going to get referenced by a "partial"

    Link to the partial by adding embedded ruby code to the HTML in ./app/views/layouts/application.html.erb

  3. Go to getbootstrap.com/components/#navbar to find the code for the navbar (this will be nested in "div" tags).

    Copy and paste a bunch of the code for the navbar in the header into the partial _header.html.erb and be sure to get the "div" tags along with everything inside.

  4. Things will still not fully work because we need to include the Bootstrap's Javascript by modifying the ./app/assets/javascripts/application.js

    Add the line of code: //= require bootstrap-sprockets

  5. Grab the "viewport" code from http://getbootstrap.com/getting-started/#template

    Copy the "meta" tags and everything inside for "viewport"

    Add a "viewport" for mobile device support at ./views/layouts/application.html.erb

    Add the viewport code to the code at the bottom of the head tag (still inside the head tags)

  6. Add a div class="jumbotron" to the top of the home page

    Go to getbootstrap.com/css/#buttons to get support for button customization

    This is done for the sign up button and is done all inside of the embedded ruby in the form of an argument.
